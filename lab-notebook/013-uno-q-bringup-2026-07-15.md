## Headless first-boot gotcha (undocumented pitfall)

The default account (`arduino`) ships with an **expired password**. On a
headless setup this blocks everything privileged тАФ `sudo`, `nmcli` WiFi
config, and SSH login all fail with "password expired" until it's reset,
and the reset is **interactive** (needs a TTY), which a scripted/
non-interactive adb session can't satisfy.

**Working method:** connect over USB and reach the board with `adb shell`;
drive the interactive `passwd` prompts through a pseudo-TTY (an `expect`
script over `adb shell -t`) to set a new password. Only then do sudo/WiFi/
SSH work. This TTY requirement isn't called out in the quick-start guides.

## Bring-up sequence that worked

1. USB-C to host тЖТ `adb devices` sees it тЖТ `adb shell` (no auth needed).
2. Reset the expired password interactively (expect-over-adb).
3. WiFi: `nmcli dev wifi connect <SSID> password <pw>` тАФ **works as the
   user, no sudo needed** (NetworkManager/polkit allows the console user).
4. Enable SSH: `arduino-app-cli system network-mode enable` тЖТ sshd comes up
   on :22.
5. Installed the host's SSH public key into `~/.ssh/authorized_keys` **via
   adb** (no password needed over adb), then key-based SSH from the host тАФ
   no password typed anywhere.
