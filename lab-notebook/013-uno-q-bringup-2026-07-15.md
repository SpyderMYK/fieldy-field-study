# 013 — Arduino UNO Q headless bring-up (rig display node)

**Date:** 2026-07-15. The UNO Q (4 GB) is the planned HDMI display node for
the performance rig (notebook 009/010). Brought up headless (no monitor).
Infra specifics (IPs, SSID, hostnames) redacted per publication rules.

## What it is [verified]

- **Debian 13 (trixie)**, kernel 6.16.7, aarch64; Qualcomm QRB2210 (4×
  Cortex-A53) + STM32U585 MCU.
- 4 cores, 3.6 GB RAM, ~9.8 GB eMMC root. **Docker 26.1.5** preinstalled.
- Dual-brain plumbing runs as systemd services: `arduino-router` +
  `arduino-router-serial` bridge the Linux side to the STM32 over a serial
  gadget; `arduino-app-cli.service` and `adbd` also running.

## Headless first-boot gotcha (undocumented pitfall)

The default account (`arduino`) ships with an **expired password**. On a
headless setup this blocks everything privileged — `sudo`, `nmcli` WiFi
config, and SSH login all fail with "password expired" until it's reset,
and the reset is **interactive** (needs a TTY), which a scripted/
non-interactive adb session can't satisfy.

**Working method:** connect over USB and reach the board with `adb shell`;
drive the interactive `passwd` prompts through a pseudo-TTY (an `expect`
script over `adb shell -t`) to set a new password. Only then do sudo/WiFi/
SSH work. This TTY requirement isn't called out in the quick-start guides.

## Bring-up sequence that worked

1. USB-C to host → `adb devices` sees it → `adb shell` (no auth needed).
2. Reset the expired password interactively (expect-over-adb).
3. WiFi: `nmcli dev wifi connect <SSID> password <pw>` — **works as the
   user, no sudo needed** (NetworkManager/polkit allows the console user).
4. Enable SSH: `arduino-app-cli system network-mode enable` → sshd comes up
   on :22.
5. Installed the host's SSH public key into `~/.ssh/authorized_keys` **via
   adb** (no password needed over adb), then key-based SSH from the host —
   no password typed anywhere.

## Rig-critical check [verified]

From the UNO Q, an HTTP GET to the message-bus host (alien's signage server)

UNO Q can drive a monitor as the rig's display node without further routing
work.
